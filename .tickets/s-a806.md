# Pre-render union type docs into MCP execute prompt + fix search

## Problem

After s-5c98, the execute prompt lists union helpers by name only ("Creation: createEntity, deleteEntity, updateEntity") and tells the agent to call search('createEntity <type>') for per-type fields. Three issues:

1. search('createEntity person') returns 'no matches' — the suggested
   discovery path doesn't work.
2. Agent has no way to learn that type:'event' needs startDate or that
   type:'job' supports companyName auto-resolve without making a call
   that fails validation.
3. The deleteEntity description says 'Types: person, education, product,
   project, technology' but actually covers all 14 union variants —
   stale subagent text.

User direction (this ticket): pre-render type docs into the prompt
*and* fix the search path. Both should give the agent the same info,
derived from the same source.

## Implementation

### Single source of truth: zod introspection

Add describeEntitySchemas() to bridge.ts. Walks
CreateEntitySchema/UpdateEntitySchema/ReviewEntitySchema (the
discriminated unions) and yields per-variant:
  { type, required: ['name', 'bio'], optional: ['website', 'github'],
    enumFields: { category: ['language', 'frontend', ...] } }

zod 4 exposes .shape and ._def.innerType cleanly; field rendering
becomes 'name: string, bio: string, website?: string, category?:
language|frontend|...'. Tested in the previous probe.

### Prompt path

server.ts buildExecuteDescription appends a 'Types' section grouped
by union:

  createEntity types:
    person — required: name, bio. optional: website, github, visible.
    event — required: title, description, link, startDate. optional: endDate, ...
    ...
  updateEntity types: person, education, ..., company, job, job-source.
    All fields except  and uid=501(jack) gid=20(staff) groups=20(staff),12(everyone),61(localaccounts),79(_appserverusr),80(admin),81(_appserveradm),33(_appstore),98(_lpadmin),100(_lpoperator),204(_developer),250(_analyticsusers),395(com.apple.access_ftp),398(com.apple.access_screensharing),399(com.apple.access_ssh),400(com.apple.access_remote_ae),701(com.apple.sharepoint.group.1) are optional.
  deleteEntity types: [all 14]. Just { type, id }.
  reviewEntity types:
    job action: approve | approve-non-technical | ...
    news action: approve | hide

Cost: ~600-800 bytes (~150-200 tokens). Acceptable given the value.

### Search path

search.ts: when the agent queries 'createEntity person', 'type:event',
'event-source', etc., return the same per-variant block instead of
'no matches'. Reuses the same describeEntitySchemas() output.

### Cleanup

- Drop the misleading 'Use search('createEntity <type>') for per-type
  field requirements' line from createEntity's description (the info
  is now in the prompt directly).
- Fix deleteEntity description to accurately list all 14 types.
- Fix updateEntity description if its type list is stale.

## Acceptance

- Execute prompt's new 'Types' section lists every union variant with
  fields, derived from the actual zod schemas.
- search('createEntity person') returns the field list.
- search('event-source') returns its create + (any) update schema info.
- Adding a new variant to a union schema automatically appears in both
  paths without touching server.ts or search.ts.
- pnpm run build / lint:fix / test --run all pass.
