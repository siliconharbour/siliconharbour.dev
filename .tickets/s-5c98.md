# Maximum consolidation: union everything except external search

## Goal

Land on the smallest possible host function count, using discriminated
unions wherever the agent-facing 'verb' is the same and the entity is
the variable. This is breaking — no backwards compat — to match the new
guiding-light approach established in s-c158.

## Final surface (~22 host functions, down from 57)

### Read (unchanged, 8 functions)
events, jobs, companies, groups, people, technologies, education, news

### Entity CRUD (5 functions)
- createEntity({ type, ...fields }) — covers ALL 9 entity types
  (company, group, event, job, person, education, product, project,
  technology) + 3 import source types (event-source, job-source,
  news-source) + 2 news content types (news-article, news-link)
- updateEntity({ type, id, ...fields })
- deleteEntity({ type, id })
- getEntity({ type, by: 'id'|'slug'|'name', value })
- listEntities({ type, filter?: 'manual'|'pending'|'all' })

### Sync (5 functions)
- syncSource({ type, sourceId })
- syncAllSources({ type? }) — type optional: scope to one source kind, or sync all
- asyncSyncAllSources({ type? })
- getAsyncSync(runId)
- listAsyncSyncs()

### Review/lifecycle (1 function)
- reviewEntity({ type, id, action }) — covers reviewJob, approveNews,
  hideNews, deactivateJob

### Importer metadata (1)
- listImporterTypes()

### External search (4 — kept as-is, they aggregate from external APIs)
- searchIndeedJobs, searchLinkedInJobs, listTechNLJobs, getTechNLJob

## Removed (the bespoke create/update/delete/getX/sync/review functions)

All ~30 of:
createCompany, createGroup, createEvent, createJob, createEventSource,
createJobSource, createNewsSource, createNewsArticle, submitNewsLink,
updateCompany, updateJob, updateJobSource, getCompanyByName,
getGroupBySlug, getJobDetail, getNewsDetail, getManualJobs,
getManualEvents, pendingEvents, pendingJobs, pendingNews,
syncEventSource, syncJobSource, syncNewsSource, syncAllEventSources,
syncAllJobSources, syncAllNewsSources, asyncSyncAllEventSources,
asyncSyncAllJobSources, asyncSyncAllNewsSources, reviewJob,
deactivateJob, approveNews, hideNews.

## Acceptance

- All capabilities still reachable through the union surface.
- searchSpec routes 'createJob' / 'syncEventSource' / etc. queries to
  the relevant union function.
- All 527 tests still pass.
- Execute prompt drops below 400 tokens.
- End-to-end smoke test (create/read/update/delete cycle for a
  technology + a news source + a sync run) all work from buildExecuteFunctions().
