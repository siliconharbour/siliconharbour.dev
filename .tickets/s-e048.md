# API leaks hidden entities through public endpoints

Several public /api/* endpoints fail to filter out non-visible/hidden entities. Detail endpoints (eg /api/companies/:slug) return entities even when visible=false. /api/events does not filter on importStatus. /api/events/:slug returns hidden imports. /api/jobs/:slug returns non-active jobs. /api/news/:slug returns drafts/hidden articles. /api/products list/detail has no visible filter (but products schema has no visible field). /api/entities/search returns non-visible companies/groups/people/education. Calendar endpoint already filters correctly via getEventsForMonth.

## Acceptance Criteria

TDD tests prove the leaks. Each endpoint is fixed to filter out hidden/non-public entities. All endpoints have failing-then-passing tests covering visibility. pnpm test passes. pnpm run build passes.
