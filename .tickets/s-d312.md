# CI: pnpm version mismatch fails Docker build (s-c09b regression)

## Root cause

The sharp patch in s-c09b ran 'pnpm patch sharp' + 'pnpm patch-commit' under pnpm 10.18.3 locally. pnpm 10 wrote two records:

- pnpm-workspace.yaml: 'sharp: patches/sharp.patch' (short form)
- pnpm-lock.yaml: 'sharp: { hash: ..., path: patches/sharp.patch }' (structured form)

The Docker build's pnpm-base layer pinned 'pnpm@9'. pnpm 9 treats those two patchedDependencies entries as structurally different, so 'pnpm install --frozen-lockfile' fails with ERR_PNPM_LOCKFILE_CONFIG_MISMATCH. CI has failed on every push since s-c09b.

## Fix

1. Dockerfile: bump 'npm install -g pnpm@9' to '@10' to match local dev.
2. package.json: pin 'packageManager': 'pnpm@10.18.3' so future contributors and corepack agree.

## Verification

- pnpm install --frozen-lockfile passes cleanly locally (we're already on 10.18.3).
- Tested in an isolated tmpdir with just manifests + patches/: fresh install succeeds, exit 0.
- All quality gates still pass.
