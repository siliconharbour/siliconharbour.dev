# MCP: full news parity with events/jobs (read, sources, sync, async)

## Why

Inventory of MCP tools showed news is the only domain missing read + source-management + sync coverage, even though there are already 6 active RSS feeds (technl, Bounce, VOCM, NTV, MUN Gazette, BetaKit) and a complete server-side news-importers/sync.server.ts.

The AI agent can already triage (`pendingNews`, `approveNews`, `hideNews`) and create one-offs (`submitNewsLink`, `createNewsArticle`) — but it can't list news, see sources, or trigger syncs. That means it can't proactively notify the human of new RSS items as the user requested.

## Acceptance

1. `news(opts)` read function on `buildReadFunctions()` mirroring `events`/`jobs`. Accepts `{ limit?, offset?, query? }`. Returns published items by default (matches public visibility).
2. `newsImportSources()` listing function mirroring `eventImportSources()` / `jobImportSources()`. Returns id, name, sourceType, sourceUrl, lastSyncAt, lastSyncStatus, lastSyncError, enabled, pendingCount.
3. `syncNewsSource(sourceId)` calling existing `syncNewsSource`.
4. `syncAllNewsSources()` iterating all enabled sources.
5. `asyncSyncAllNewsSources()` using the existing `startAsyncSync` infra.
6. `asyncSyncAllSources()` extended to include news alongside events and jobs.
7. `createNewsSource({ name, sourceType, sourceUrl, sourceIdentifier?, keywords?, useGlobalKeywords?, excerptMode?, entityUrl? })` calling existing `createNewsImportSource`.
8. `getNewsDetail(id)` returning the full news row (parallel to `getJobDetail`).
9. `async-syncs.ts` type union extended from `'event' | 'job'` to `'event' | 'job' | 'news'`.
10. `server.ts` system prompt updated to list every new function and note that news sources are mostly NL RSS feeds the agent can sync to find recent posts.
11. `pnpm run build`, `pnpm run lint:fix`, `pnpm test --run` all pass.

## Out of scope

- No new news source types (rss/custom is enough for now).
- No `updateNewsSource` / `deleteNewsSource` (admin UI exists for that).
- No news reviews like `reviewJob` action enum — `approveNews` and `hideNews` already cover the lifecycle.
