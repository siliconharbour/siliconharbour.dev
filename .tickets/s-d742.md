# Generate event cover image from icon palette

## Why

Not every event has a good banner image, but most have an icon (org logo,
square thumbnail). Need a way to synthesize a 1200x400 cover from the
icon's dominant colors so admins can publish without sourcing or
designing a banner.

## What

Server-side palette extraction + gradient synthesis using sharp (already
installed). Approach: diagonal 2-color gradient with subtle noise.

1. New helper app/lib/images.server.ts: generateCoverFromIcon(iconBuffer)
   - Extract two dominant colors via sharp downscale + .raw() pixel
     sampling + simple bucket histogram (avoid extra deps).
   - Compose 1200x400 SVG with linearGradient between them on the
     diagonal, plus a feTurbulence noise overlay at low opacity.
   - Pipe through sharp.png().toBuffer() then the existing
     processAndSaveCoverImage so it matches the rest of the system
     (resize/webp/uuid filename).

2. EventForm: new checkbox 'Generate cover from icon' under the cover
   image slot. Disabled when no icon is selected/present. When ticked,
   the cover upload slot collapses into a placeholder explaining what
   will happen on save.

3. Action handlers (new.tsx + edit.tsx): when formData has
   generateCoverFromIcon=true, prefer the just-uploaded icon buffer,
   fall back to the saved icon file on disk, run generateCoverFromIcon,
   save as the cover. Skip any cover upload sent in the same form.

4. Cleanup: if there was an old cover and we're replacing it with a
   generated one, delete the old file (matches resolveUpdatedImage).

## Acceptance

- Editing an event with an icon: tick 'Generate cover from icon' -> save -> reload -> see a gradient cover using the icon's colors.
- Creating an event with both icon upload and 'Generate cover from icon' ticked: cover is synthesized from the icon being uploaded in the same submission.
- Toggling off the checkbox restores the regular upload UI.
- Existing cover upload path keeps working unchanged.
- pnpm run build, lint:fix, test --run all pass.
