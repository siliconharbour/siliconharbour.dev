# MCP test coverage: sandbox isolation + bridge union smoke + introspection

## Why

Commit 8ad957a fixed a production bug where the QuickJS sandbox was loaded once at module init and reused across all calls — poisoned state from one query could affect subsequent ones. The fix creates a fresh runner per invocation.

The pre-fix test commit (4559bd9) added two simple round-trip tests but neither would have caught the bug because they only ran a single query each. We have no tests covering:

1. Sandbox isolation across calls (the bug class).
2. Any of the bridge.ts host functions (2058 LOC, zero direct tests).
3. The introspection helpers (getEntitySchemaDocs, getHostFunctionDocs) that drive /api page and execute prompt.

## What

### Sandbox isolation tests (would-have-caught-the-bug)
- Sequential calls don't share globals or runtime state.
- A throwing call doesn't poison the next clean call.
- Concurrent calls don't interfere with each other.
- (Bonus) Long sequence — N=10 calls in a row, each independent.

### Bridge union dispatch smoke tests
One happy-path per discriminated union:
- createEntity({ type:'person' }) / createEntity({ type:'company' }) / createEntity({ type:'news-link', ... auto-fetch })
- updateEntity({ type:'company', id, ... })
- deleteEntity({ type:'event-source', id })
- getEntity({ type:'company', by:'name', value })
- listEntities({ type:'event', filter:'manual' })
- reviewEntity({ type:'job', action:'approve' })
- syncSource / syncAllSources / asyncSyncAllSources (one call each, mocked underlying sync)

### Introspection helper tests
- getEntitySchemaDocs() returns expected variant counts (14 create, 8 update, 2 review).
- Specific variants have expected required + optional fields.
- Enum fields render their values (e.g. technology category includes 'language').
- Output is deterministic across calls.
- getHostFunctionDocs() lists 27 execute functions with no 'undocumented' entries.

## Acceptance

- All new tests pass.
- Sandbox isolation tests fail when run against the pre-fix sandbox.ts (manual verification).
- Test suite total runtime stays under 60s.
- pnpm run build / lint:fix / test --run all green.

## Out of scope

- Per-host-function comprehensive testing (the larger 50-test option).
- Lib-level coverage for events.server.ts, github.server.ts, etc. — separate concern.
- Integration tests against a real MCP client.
