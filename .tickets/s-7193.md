# Discord: fix getBotMember - @me not valid for guildMember route

Routes.guildMember(guildId, '@me') sends to /guilds/{id}/members/@me which Discord rejects with NUMBER_TYPE_COERCE because the route validator expects a snowflake. Discord does have GET /guilds/{guild.id}/members/@me but it's a different route path than the parameterised one. Solution: resolve the bot's user id from GET /users/@me first (cached), then call Routes.guildMember(guildId, botUserId).

## Acceptance Criteria

Adding a channel destination in a guild where the bot is NOT an administrator no longer 400s. New test (or extension to existing test) confirms getBotMember works. The cached user id is reused across guild lookups so we don't hit /users/@me repeatedly.
